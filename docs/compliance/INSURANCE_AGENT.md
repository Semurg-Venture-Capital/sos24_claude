# INSURANCE_AGENT.md — Соответствие «Положению о страховых агентах»

> **Документ:** «Суғурта агентлари тўғрисидаги Низом» (Положение о страховых агентах),
> рег. № **3845**, **02.06.2026**.
> **Зачем нам:** SOS24 — онлайн-платформа, оформляющая страховые полисы как **страховой
> агент** (в Положении прямо перечислены такие платформы: webpolis.uz, sugurtabozor.uz,
> Road24, Click, Zorgo и др.). Соблюдение Положения — **лицензионное требование к
> страховщику**; за нарушение регулятор применяет меры к страховщику, а тот — к нам по
> агентскому договору.
>
> **Что обязаны соблюдать (для онлайн-платформ):** пп. **3, 4, 5, 6, 10, 11, 12, 15, 16**.
> Открытые вопросы к заказчику — `docs/QUESTIONS.md` раздел 13 (Q13.1–Q13.6).

---

## 1. Кто мы по Положению

- **Страховой агент** — посредник, организующий заключение и исполнение договоров страхования.
- SOS24 действует через **собственное ПО (информационную систему)** → подпадает под усиленные
  требования к платформам (пп. 3, 11).
- **Единая автоматизированная информационная система (ЯААТ)** — государственная система учёта
  рынка страхования, в которую передаются данные о договорах и регистрируются агенты.
  *(Уточняется, тот же ли это шлюз, что НАПП/e-osgo — см. Q13.2.)*

## 2. Требования к нам (по пунктам Положения)

| Пункт | Требование | Что значит для SOS24 |
|---|---|---|
| **3** | Посредничество в электронной форме через ПО страховщика **или своё**. Данные о заключённых в нашем ПО полисах **обязаны передаваться в ЯААТ через интеграцию** (по её запросу). | **Техинтеграция с ЯААТ**: после оформления/по запросу слать данные полисов. Самый крупный техпункт. |
| **4** | Агент работает **только после** агентского договора (агент битими) и внесения в реестр страховщика. Страховщик за **1 день** регистрирует договор в ЯААТ → агенту выдаётся **ID агента**. Расторжение — отметка за 1 день. | Нужен **агентский договор** + хранить **agentId** и статус договора на каждую СК. |
| **5** | (2-й абзац) Особенности, когда юрлицо-агент сам выступает страхователем в договоре. | Если SOS24 страхует собственные риски — отдельный учёт; в обычном посредничестве не применимо. |
| **6** | Функции агента нельзя выполнять по **иному** гражданско-правовому договору, кроме агентского. | Оформляем отношения со страховщиком только агентским договором. |
| **7** | **Кредитные** страховки: агент — банк-кредитор (его филиалы), через ПО банка; страхователь свободно выбирает страховщика; только банк-выгодоприобретатель может быть агентом. | Касается нас только при страховании, привязанном к банковским кредитам (см. Q13.6). |
| **10** | Страховщик обучает агента курсом **≥12 часов**; у юрлица-агента в штате — **минимум 1 обученный сотрудник**. | Обеспечить обучение ≥1 сотрудника. |
| **11** | Агент, работающий через своё ПО, обязан **непрерывно** вести **поддержку клиентов** (call-центр, интерактивный чат и т.п.). | Постоянная служба поддержки (наш ИИ-чат/колл-центр — теперь обязателен). |
| **12** | При оформлении: соблюдать **мин. условия договора** (уполномоченного органа) в интересах страхователя; **знакомить клиента с официальным рейтингом страховщиков**. | Показывать в UI мин. условия + рейтинг СК. |
| **15** | Комиссия (воситачилик хаки) — **не более 25%** премии, поступившей на счёт страховщика. Запрещено платить сотрудникам страховщика за посредничество по агентскому договору. | Лимит комиссии **≤25%**; учесть в финрасчётах. |
| **16** | При **досрочном расторжении** полиса комиссия возвращается страховщику **пропорционально** неистёкшему сроку. | Логика пропорционального возврата комиссии. |
| **17** | Права/обязанности по агентскому договору и посреднические услуги по нему **нельзя передавать третьим лицам**. | Без субподряда/передачи функций агента. |

## 3. Влияние на архитектуру (наш insurance-модуль)

Уже есть: каталог `InsuranceCompany` / `InsuranceProduct` / `ProductPlan`, оформление полиса
с привязкой `companyId/productId/planId` (см. `docs/STAGE1.md`). Чего не хватает под Положение:

- [ ] **Агентский договор и ID агента на компанию** — поля у `InsuranceCompany`
      (напр. `agentId`, `agreementStatus`, `agreementFrom/To`); оформлять полисы только
      по компаниям с активным договором (п.4, п.6).
- [ ] **Интеграция с ЯААТ** (п.3) — модуль отправки данных заключённого полиса; повторная
      отправка по запросу; статус синхронизации у полиса. **Нужен доступ/спецификация API** (Q13.2).
- [ ] **Рейтинг страховщиков** (п.12) — поле/источник рейтинга у компании + показ в карточке
      выбора компании и при оформлении.
- [ ] **Мин. условия договора** (п.12) — справочник минимальных требований по видам страхования,
      показ/валидация при оформлении.
- [ ] **Комиссия ≤25%** (п.15) — поле комиссии у продукта/компании с верхней границей 25%;
      **пропорциональный возврат** при досрочном расторжении (п.16) в финансовой логике.
- [ ] **Служба поддержки** (п.11) — непрерывный канал (ИИ-чат/колл-центр) — уже в планах.
- [ ] **Обучение сотрудника ≥12ч** (п.10) — организационное, не код.

См. `docs/QUESTIONS.md` раздел 13:

- **Q13.3** — кто проходит 12-часовой курс.
- **Q13.4** — источник/формат рейтинга страховщиков и минимальных условий договора.
- **Q13.5** — конкретный размер комиссии (≤25%) и механика расчётов/возврата.
- **Q13.6** — делаем ли кредитные страховки (п.7).
