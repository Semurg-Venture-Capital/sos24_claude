# SOS_ASSISTANT.md — SOS-кнопка · ИИ-помощник (центральный LLM-роутер)

> **Что это.** SOS-кнопка на главном открывает **ИИ-ассистента «SOS24 · ИИ-помощник»** —
> центральный LLM-роутер платформы: разбирается в ситуации и ведёт пользователя в нужный
> сценарий (европротокол, помощь на месте, мед-триаж, покупка полиса, звонок специалиста),
> а также помогает ориентироваться в приложении.
>
> **Дизайн-эталон (утверждён):** `SOS24/screens-sos.jsx` → `ScreenSosChat` (компоненты
> `SosAiCard`, `SosCategoryRow`, `SosAiAction`, `SosUserBubble`, композер с камерой).
> **Статус:** решение согласовано (2026-07-15), код не начат.

---

## 1. Решения (согласованы с заказчиком)

| Развилка | Решение |
|---|---|
| Поведение SOS-кнопки | **ИИ-чат** + постоянная кнопка **«Экстренный звонок»** (1 тап при угрозе жизни). Ассистент ведёт, но НЕ тормозит экстренность. |
| Человек-хендофф | **Заявка на callback** — кнопка создаёт заявку (телефон + контекст чата) → операторы центра поддержки перезванивают. Переиспользуем поддержку/колл-центр. |
| Объём v1 | **Ядро:** чат + категории + роутинг в готовые флоу + экстренный звонок. Callback — Фаза 2. |

## 2. UX (по дизайну)

Экран-чат:
- **Хедер:** назад · SOS-лого (красный градиент) с зелёной точкой «на связи» · «SOS24 · ИИ-помощник» · статус · кнопка «Экстренный звонок» (phone).
- **Диалог:** системная плашка (защищено) → приветствие ИИ (персональное) → **категории-триаж** (`SosCategoryRow`): ДТП · Мед. помощь · Угон/кража · Имущество · Другое → пузыри пользователя (`SosUserBubble`) → **действия ИИ** (`SosAiAction`): напр. «Оформить европротокол», «Вызвать помощь на месте», «Связать со специалистом».
- **Композер:** «Заказать звонок — ответим за 30 сек» (Фаза 2) · камера (Фаза 3) · поле «Опишите ситуацию…» · отправка.

**Экстренность:** при высокой срочности (травмы, пожар, угроза жизни) ассистент СРАЗУ выносит наверх блок «Экстренный вызов» (103/101/102/112) — до долгого чата.

## 3. Архитектура

### 3.1. Бэкенд — новый модуль `apps/api/src/assistant/`
- Переиспользует **`LlmService`** (Gemini, в проде) + лог `AiUsageLog`.
- **LLM-роутер со структурным выводом** (шаблон — `health/triage/triage.provider.ts`):
  вход — история сообщений + локаль; выход (JSON responseSchema):
  ```
  {
    reply: string,            // ответ ассистента (на языке пользователя)
    category: 'accident' | 'medical' | 'theft' | 'property' | 'insurance' | 'other',
    urgency: 'low' | 'medium' | 'high',
    actions: Action[]         // из ЗАКРЫТОГО набора (см. 3.2)
  }
  ```
- **Принцип безопасности:** LLM только КЛАССИФИЦИРУЕТ и ПРЕДЛАГАЕТ действия из закрытого списка; **навигацию выполняет приложение** — модель никогда сама не «переходит» и не вызывает службы.

### 3.2. Таксономия действий (Action) — что LLM может предложить → куда ведёт
| action | Назначение | Куда ведёт в приложении |
|---|---|---|
| `europrotocol` | оформить европротокол | `EuroStart` |
| `onsite_help` | вызвать помощь на месте | `AdjusterRequest` |
| `health_triage` | мед-консультация | `HealthTriage` (существующий триаж) |
| `emergency_call` | экстренный звонок | `tel:103/101/102/112` |
| `request_callback` | заказать звонок | заявка на callback (Фаза 2) |
| `buy_policy` | оформить/подобрать полис | Каталог/продукт |
| `support` | связаться с поддержкой | Поддержка (тикет/чат) |
| `navigate` | «покажи мои полисы/гараж…» | deep-link в раздел (ориентирование) |

Каждый `Action` = `{ type, label, hint?, params? }`. Приложение рендерит их как `SosAiAction` и знает, куда вести (жёсткий маппинг type→route на клиенте).

### 3.3. Мобилка — `apps/mobile/src/features/sos/`
- Экран-чат по дизайну (переиспользуем UI-паттерны из `HealthTriageScreen` + компоненты из `screens-sos.jsx`).
- **`SosBanner` на Home → этот чат** (сейчас не привязан — гринфилд).
- Категории-чипы (быстрый старт) + free-text.
- Persist сессии (как `triageStore`), чтобы диалог не терялся.
- **Мед-ветка отдаёт в существующий триаж** (`HealthTriage`) — не дублируем медицину.

### 3.4. Данные — `AssistantSession` (Prisma)
- `userId`, `messages` (ШИФРУЮТСЯ — как `TriageSession`), `category`, `urgency`,
  `linkedEuroId`/`linkedAdjusterId`/`linkedCallbackId` (связанные сущности), `createdAt/updatedAt`.
- Каждый вызов LLM → `AiUsageLog` (feature='assistant_route').

## 4. Безопасность и экстренность (это «экстренная помощь»)
- Кнопка **«Экстренный звонок»** всегда видна (хедер) — 1 тап.
- Строгий системный промпт: не ставит диагнозов, не даёт юр. заключений; при угрозе жизни — сразу к экстренным службам/человеку; кратко в экстренных ситуациях.
- Дисклеймер: «Не заменяет экстренные службы 103/112».
- Экстренные номера РУз (проверить): **103** скорая · **101** пожарная · **102** полиция · **112** единый.

## 6. Фазы
- **v1 — Ядро:** чат + LLM-роутер + категории + роутинг в готовые флоу + экстренный звонок. Привязать `SosBanner`.
- **v2 — Хендофф:** «Заказать звонок» → заявка на callback операторам (поддержка/колл-центр) + при желании переход в чат поддержки.
- **v3 — Богатый:** фото с места (MinIO → в европротокол), голосовой ввод, проактивный контекст (гео, активные полисы/авто).
