# PDF_FILL.md — Автозаполнение бланка Европротокола (УЗ)

> Как мы программно заполняем официальный бланк «Yo'l-transport hodisasi to'g'risida xabarnoma».
> Статус (2026-06-10): **калибровка приостановлена частично готовой** — решено сперва нарисовать экраны Европротокола, генерацию PDF и доводку координат делаем после.

## Текущий статус калибровки (2026-06-10)

**Готово / проверено:**
- Харнесс: `grid.mjs` (сетка), `testfill.mjs` (тест-заполнение), `detect.py` (пиксель-детектор клеток).
- Шаг comb-клеток исправлен **12.8 → 7.5pt** (полноширинный ряд ≈30 клеток).
- `make_model` — это `text` (две линии), не comb. Раздел регистрации разделён (гос.рег.номер + свид-во серия/номер).
- Кириллица рендерится (Manrope TTF + fontkit).

**Осталось (доводка координат):**
- Системный сдвиг baseline ≈ +8pt у comb-рядов (текст садится на лейбл, клетки ниже) — опустить `y`.
- Нижние поля колонки (телефон, ВУ, страховщик, полис, даты) + вся страница 2 — координаты черновые.
- **Блокер точности:** перегородки клеток в скане бледные/рваные → для надёжной автоэкстракции нужен `numpy`+`opencv` (не установлены) ИЛИ ручные визуальные итерации.

**Решение:** сначала экраны (Вариант A, 11 экранов), затем вернуться к генерации PDF + доводке `fieldmap`.

## Факты о бланке (критично)

- **Плоский PDF**: нет текстового слоя, нет AcroForm-полей, нет встроенных шрифтов (`Form: none`, `pdffonts` пуст).
- Заполнение **только наложением** по координатам (`pdf-lib.drawText` / `drawImage`).
- **Шрифт обязательно встраивать** (Inter/Mont TTF + `@pdf-lib/fontkit`) — иначе кириллица/латиница не отрисуется.
- Страница A4: **595.276 × 841.89 pt**. Origin pdf-lib — **внизу-слева**, Y растёт вверх (на скане Y инвертирован!).

## Система координат и типы полей

Все координаты в `fieldmap.draft.json` — в **точках PDF**, origin внизу-слева. Типы:

- **text** — строка по `(x, y)` baseline, перенос по `maxWidth`.
- **comb** — посимвольно в клетки: `x` (первая клетка) + `pitch` (шаг) + `cells` (число). ФИО, госномер, № кузова/двигателя, серии/номера, даты.
- **checkbox** — «×» по `(x, y)` нужного квадрата (Ҳа/Йўқ, КАСКО, владелец, 22 обстоятельства).
- **marks** — обвести один из преднапечатанных вариантов (категория ВУ).
- **image** — картинка в прямоугольник: схема ДТП, зона удара, подпись, фотополоса.
- **static** — преднапечатанный текст, не заполняем.

## Структура бланка

**Стр.1 (лицевая):** общая часть (пп.1–6) · колонка «А» (x≈14–248) · центр — 22 обстоятельства с чекбоксами A/B (x≈250–348) · колонка «В» (x≈352–582) · «15» схема ДТП · «16» подписи.
**Стр.2 (оборотная):** «7» обстоятельства текстом · 7.4/7.5 чекбоксы · «8» замечания · фотозона · подписи · инструкция (static).

⚠️ На реальном бланке **~22 обстоятельства**, а не 17 (как в `EUROPROTOCOL.md §6`).

## Что заполняем автоматически (сторона «А»)

| Поле бланка | Источник |
|---|---|
| Марка/модель, № кузова/двигателя, госномер | `Vehicle` (+НАПП) |
| Владелец ФИО/адрес | MyID-профиль |
| Водитель ФИО/ДР/ВУ серия-номер/категория | Documents + MyID |
| Телефон | `User` |
| Страховщик, полис серия/номер, срок | `Policy` |
| Дата/время/место/GPS, медосвидетельствование | данные инцидента (экран) |

Сторона «В»: госномер → НАПП, остальное вручную (публичного реестра полисов в УЗ нет). Обстоятельства/схема/повреждения/признание вины — ввод на экранах.

## Воркфлоу калибровки (как выверить координаты)

1. Сгенерировать сетку: `node grid.mjs` (см. скрипт ниже) → `grid.pdf`.
2. Снять черновые координаты с `grid-full-*.png` → `fieldmap.draft.json` (сделано).
3. **Тест-рендер**: залить во все поля маркеры (id поля / «XXXX») и отрисовать поверх бланка → PNG.
4. Сравнить с бланком, подвинуть координаты/pitch, повторить 3–4 до попадания.
5. Зафиксировать `calibrated: true`, перенести карту в генератор (`apps/api`).

Скрипт сетки (одноразовый): `/tmp/evrocal/grid.mjs` (pdf-lib). При необходимости перенесём в репо.

## Технологии (self-hosted, без облака)

- `pdf-lib` + `@pdf-lib/fontkit` — заполнение/наложение на бэке (`apps/api`).
- `poppler` (`pdftoppm`/`pdfinfo`/`pdffonts`) — рендер/инспекция при разработке.
