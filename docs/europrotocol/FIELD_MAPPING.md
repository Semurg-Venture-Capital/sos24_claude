> Обновлено: 2026-06-15. Связано: [PDF генерация](../../apps/api/src/europrotocol/pdfgen/), `EUROPROTOCOL.md`, `PDF_FILL.md`.

## Модель документа

**Один PDF на двоих.** Стр.1 поделена пополам: колонка **«А»** — инициатор (наш пользователь),
колонка **«В»** — второй участник. Стр.2 (оборот) — пока заполняем со стороны «А» (см. открытый вопрос внизу).

Источники, которые у нас есть:
- `EuroProtocol` (incident + сторона B raw), `EuroParticipant` (MyID 2-го), `User`+MyID (сторона A),
  `Vehicle` (+НАПП), `Document` (паспорт/ВУ), `Policy` (ОСАГО/КАСКО).

Легенда статусов: ✅ есть · ⚠️ частично (надо достать/распарсить/маппить) · ❌ нет (собирать в приложении или добавить в модель).

## 1. Общая часть (стр.1, пп.1–6) — заполняется совместно

| Поле бланка | Статус | Источник / что делать |
|---|---|---|
| 1. Место ДТП (place) | ✅ | `EuroProtocol.place` |
| 2. Дата ДТП (DD MM YYYY) | ✅ | `EuroProtocol.incidentDate` |
| 2. Время (соат, дақиқа) | ✅ | `EuroProtocol.incidentTime` |
| 3. Число пострадавших Т/в | ⚠️ | по смыслу всегда `2` (европротокол = 2 ТС) → константа |
| 4. Медосвидетельствование пройдено (Ҳа/Йўқ) | ❌ | **нет в модели** → добавить `medCheck` + экран |
| 5. Свидетели (ФИО, адрес) | ❌ | **нет** → опц. поле визарда `witnesses[]` |
| 6. Оформлено сотрудником ИИО/ДЙҲХ (Ҳа/Йўқ) | ❌ | **нет** → поле `officialRegistered` (обычно «Йўқ») |
| 6. № нагрудного знака (Кўкрак нишони рақами) | ❌ | **нет** → поле `officerBadgeNo` (если вызывали ГАИ) |

## 2. Колонка «А» (инициатор) — данные из системы

| Поле бланка | Статус | Источник / что делать |
|---|---|---|
| 7. Марка/модель | ✅ | `Vehicle.brand` + `model` (+`year`) |
| № кузова/шасси | ✅ | `Vehicle.bodyNumber` (НАПП) ∥ `vin` |
| № двигателя | ✅ | `Vehicle.engineNumber` (НАПП) |
| Госномер (давлат рўйхати) | ✅ | `Vehicle.plate` |
| Свид-во о рег.: серия/номер (техпаспорт) | ✅ | `Vehicle.techPassportSeria` / `techPassportNumber` |
| 8. Владелец ТС (ФИО) | ⚠️ | `Vehicle.ownerName` (НАПП) ∥ `User` ФИО, если владелец = пользователь |
| 8. Адрес владельца | ⚠️ | `User.address` (MyID) — **верно только если владелец = пользователь**; иначе адрес владельца ❌ |
| 9. Водитель (ФИО) | ✅ | `User.surname/name/patronymic` |
| 9. Дата рождения | ✅ | `User.birthDate` (MyID) |
| 9. Адрес водителя | ✅ | `User.address` (MyID) |
| Телефон | ✅ | `User.phone` |
| ВУ: серия/номер | ✅ | `Document(DRIVER_LICENSE).series` / `number` |
| Категория (A B C D BE CE DE) | ✅ | `Document(DRIVER_LICENSE).categories` (CSV) |
| ВУ: дата выдачи | ✅ | `Document(DRIVER_LICENSE).issuedAt` |
| Документ о праве владения (если водитель ≠ владелец) | ❌ | **нет** → доверенность/договор: поле `ownershipDoc` |
| 10. Страховщик | ⚠️ | имя нашей СК (константа/конфиг) — в `Policy` названия нет |
| Полис: серия/номер | ⚠️ | `Policy.policyNumber` есть, но **не разбит** на серию+номер → распарсить или хранить отдельно |
| Полис: срок действия | ✅ | `Policy.endDate` |
| КАСКО (Ҳа/Йўқ) | ⚠️ | вычислить: есть ли у пользователя активный `Policy(type=KASKO)` на это ТС |
| 11. Место первого удара (стрелка на схеме) | ❌ | **нет** → экран: отметка зоны на схеме авто → image |
| 12. Описание повреждений | ⚠️ | `EuroProtocol.description` общий; **по стороне нет** → поле `damageDescA` |
| 13. Эътирозлар (возражения) | ❌ | **нет** → поле `objectionsA` |
| Подпись водителя «А» | ❌ | **нет** → OTP/MyID-штамп или рисованная подпись → image |

## 3. Колонка «В» (второй участник)

| Поле бланка | Статус | Источник / что делать |
|---|---|---|
| 7. Марка/модель | ⚠️ | `EuroProtocol.otherVehicleRaw` (НАПП TechPassportInfo) |

| 9. Водитель (ФИО) | ⚠️ | `EuroParticipant` (MyID 2-го, по ПИНФЛ) ∥ ручной ввод |
| 9. Дата рождения | ⚠️ | `EuroParticipant.birthDate` |
| 9. Адрес водителя | ⚠️ | `EuroParticipant.address` (MyID) |
| Телефон | ✅ | `EuroProtocol.otherPhone` |
| ВУ: серия/номер | ❌ | **нет** (у participant паспорт, не ВУ) → ручной ввод `otherDl*` |
| Категория ВУ | ❌ | **нет** → ручной ввод |
| ВУ: дата выдачи | ❌ | **нет** → ручной ввод |
| Документ о праве владения | ❌ | **нет** → ручной ввод |
| 10. Страховщик (название СК 2-го) | ❌ | **нет** → ручной ввод `otherInsurer` |
| Полис: серия/номер | ✅ | `otherPolicySeria` / `otherPolicyNumber` |
| Полис: срок действия (дата) | ❌ | есть только `otherPolicyValid` (Boolean «действует»), **даты нет** → ручной ввод |
| КАСКО (Ҳа/Йўқ) | ❌ | **нет** → ручной ввод (обычно «Йўқ») |
| 11. Место удара / 12. Повреждения / 13. Эътирозлар | ❌ | **нет** → ввод на экранах (для стороны B) |
| Подпись водителя «В» | ❌ | **нет** → OTP по `otherPhone` / QR-ссылка (см. EUROPROTOCOL.md §9.1) |

### A. Новые поля в `EuroProtocol` (общие/сторона A/оборот)
- `medCheck` (bool), `witnesses` (json/text), `officialRegistered` (bool), `officerBadgeNo` (string?)

- `ownershipDocA` (string?) — документ о праве владения А

- `signatureA` (string/url — image), `signatureB` (string/url), `schemeImage` (string/url), `impactPointA`/`impactPointB` (string/url или код зоны)

- Счётчики отмеченных обстоятельств.

### D. Артефакты-изображения (нужен пайплайн загрузки в MinIO)
