# WHOOP — инструкция по получению ключей и настройке приложения

> Пошаговое руководство: как создать приложение WHOOP в дашборде разработчика, получить
> `client_id`/`client_secret`, настроить redirect и вебхуки под SOS24.
>
> Технический контекст — `docs/integrations/WHOOP.md`. Код интеграции уже готов (mock + real).

---

## 0. Что понадобится заранее

- **Аккаунт WHOOP** (обычный, тот же что в приложении WHOOP).
- Для проверки **реальных данных** — физический **браслет WHOOP + активная подписка** (WHOOP
  требует, чтобы у разработчика было устройство). Создать приложение можно и без браслета,
  но данные придут только с реального устройства.
- До одобрения приложения интеграцией могут пользоваться **максимум 10 человек**. Для публичного
  запуска нужно пройти ревью WHOOP (раз в месяц).

---

## 1. Данные SOS24 для заполнения (скопировать)

| Поле | Значение |
|---|---|
| **App Name** | `SOS24` |
| **Contact Email** | (рабочая почта команды) |
| **Privacy Policy URL** | `https://api.sos24.uz/privacy` *(наша страница политики; позже продублируем на sos24.uz)* |
| **Redirect URI** | `https://api.sos24.uz/health/wearable/whoop/callback` |
| **Webhook URL** | `https://api.sos24.uz/webhooks/whoop` |
| **Scopes** | `offline`, `read:recovery`, `read:sleep`, `read:cycles`, `read:body_measurement`, `read:profile` |

> `offline` обязателен — без него не выдаётся refresh-token (данные перестанут обновляться через час).

---

## 2. Пошагово

1. Открой **https://developer-dashboard.whoop.com** и войди под своим аккаунтом WHOOP.
2. Создай **Team** (команду) — требуется перед первым приложением. Название любое, напр. `SOS24`.
3. Нажми **Create App** (`https://developer-dashboard.whoop.com/apps/create`).
4. Заполни поля из таблицы §1:
   - **App Name**, **Contact Email**, **Privacy Policy URL**.
   - **Scopes** — отметь все 6 из таблицы.
   - **Redirect URIs** — добавь `https://api.sos24.uz/health/wearable/whoop/callback`
     (можно добавить и второй, для локальной разработки — скажи мне, если нужно).
   - **Webhook URL** (если поле есть на этом шаге) — `https://api.sos24.uz/webhooks/whoop`.
5. Нажми **Create**.
6. Открой созданное приложение в разделе **Apps** — там будут **Client ID** и **Client Secret**.

> ⚠️ **Client Secret** — секретный, только на сервер. Не выкладывать в чат/репозиторий/скриншоты
> публично. Пришли его мне защищённым каналом (или добавим прямо в секрет прода).

---

## 3. Что прислать мне

- **Client ID**
- **Client Secret**
- Подтверждение, что в приложении заданы **Redirect URI** и **Webhook URL** из §1.

Дальше я:
1. Пропишу в секрет прода `WHOOP_MODE=real`, `WHOOP_CLIENT_ID`, `WHOOP_CLIENT_SECRET`.
2. Добавлю OAuth-браузер в мобильное приложение (одна нативная пересборка dev-client).
3. Сверю пути v2-эндпоинтов и проверю реальный обмен токенов + вебхук с боевой подписью.
4. Переключим режим mock → real — код уже готов, меняется только конфигурация.

---

## 4. Публичный запуск (позже)

Когда протестируем на своих браслетах (≤10 человек) — подаём приложение на **одобрение** в дашборде
(App Approval). Требования: заполненные App Name / Contact Email / Privacy Policy URL, тест хотя бы
с одним участником WHOOP, соответствие бренд-гайдам WHOOP. Ревью — примерно раз в месяц. После
одобрения лимит 10 пользователей снимается.
