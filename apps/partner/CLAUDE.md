# apps/partner — SOS24 Partner Cabinet (B2B)

> Этот файл дополняет корневой `CLAUDE.md` контекстом, специфичным для партнёрского кабинета. Корневой читается первым, потом этот.

## Что это

Веб-кабинет для **B2B-партнёров**: СТО и медклиники (и потенциально другие категории в будущем). Здесь партнёр:
- Заводит свою организацию и услуги
- Получает заявки от клиентов страховой (направления на ремонт, осмотры)
- Видит свой рейтинг и отзывы
- Управляет финансами (поступления от страховой, акты)
- Загружает фото/документы для подтверждения работ

## Стек

- Тот же что у `apps/admin`: Next.js 15 + Tailwind + shadcn/ui + TanStack Query + RHF/Zod
- Отличия: упрощённый интерфейс, меньше ролей (партнёр-владелец, партнёр-менеджер, сотрудник)
- Авторизация через ту же `apps/api`, но отдельный набор ролей и API-эндпоинтов

## Команды

```bash
pnpm dev:partner                      # next dev
pnpm --filter partner run build
pnpm --filter partner run typecheck
```

## Структура

```
apps/partner/src/app/
├── (auth)/
├── (cabinet)/
│   ├── orders/                # заявки от страховой
│   ├── services/              # каталог услуг партнёра
│   ├── finance/               # выплаты, акты
│   ├── profile/               # данные организации
│   └── settings/
└── api/
```

## Что специфично

- **Партнёр видит ТОЛЬКО свои данные** — строгая изоляция на уровне API (multi-tenancy)
- **Загрузка документов** — MinIO через подписанные URL
- **Уведомления** — внутри кабинета + email + push в будущем (если будет мобильное приложение партнёра)
