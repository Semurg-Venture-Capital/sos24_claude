## 0. О проекте (коротко)

Платформа автострахования **SOS24** (ОСАГО + КАСКО) для Узбекистана — позиционирование «помощник 24/7 на дороге»: продажа полисов, e-полис с QR, заявление о ДТП, вызов аварийного комиссара, партнёрская сеть (СТО, медклиники).

- **Mobile (React Native):** клиентское приложение iOS + Android
- **Web Landing (sos24.uz):** маркетинговый сайт
- **Web Admin:** внутренняя панель для сотрудников страховой
- **Web Partner Cabinet (новое):** B2B-кабинет для партнёров (СТО, медклиники)
- Языки: **узбекский латиница (uz-Latn), узбекский кириллица (uz-Cyrl), русский, английский** — **4 локали**, переключатель должен показывать все 4

## 2. Брендинг (стартовые ориентиры)

> Финальные референсы клиент уже передал. Эта секция — наша интерпретация ожиданий.

- **Имя бренда:** SOS24 — намекает на «помощь 24/7». Логотип/палитра — за дизайнером.
- **Тон:** современный, доверительный, fintech-уровня (а не «олдскульная страховая»), но с акцентом на «спасение в сложной ситуации» (название обязывает).
- **Аудитория:** автовладельцы 22–55 лет, Узбекистан, смешанные языки
- **Хорошие референсы для вдохновения:** Revolut, Tinkoff Insurance, Lemonade, Apex Insurance UZ, Gross Insurance UZ
- **Что важно:** читаемость, доступность (WCAG AA), хорошее ощущение скорости (skeleton-лоадеры, плавные переходы), узнаваемая «тревожная кнопка» (заявление о ДТП / вызов комиссара)
- **Чего избегать:** перегруженность, мелкий шрифт на формах, агрессивные «купи сейчас» баннеры

## 3. Mobile App — экраны

### M1. Онбординг + Auth
- Splash (логотип SOS24, лоадер)
- Онбординг 1: «Покупка полиса в 2 минуты»
- Онбординг 2: «Электронный полис с QR»
- Онбординг 3: «Заявление о ДТП и вызов комиссара прямо с места»
- **Выбор языка (4 варианта): O'zbek (Lotin) / Ўзбек (Кирилл) / Русский / English**
- Ввод номера телефона (+998)
- Ввод SMS-кода (4 или 6 цифр — уточнить)
- Резерв: «не пришёл код» / «изменить номер»
- (Фаза 1) Альтернативный вход через **MyID** / **OneID** — кнопки на экране
- Согласие с условиями (чекбокс + ссылки)

### M2. Профиль
- Главная профиля: аватар, ФИО, телефон, меню
- Редактирование персональных данных
- Паспорт (серия + номер + скан/фото)
- Водительское удостоверение (номер + категория + срок + скан) — позже автозаполнение через NAPP
- Адрес проживания
- Настройки: **язык (4 варианта)**, тема (light/dark), уведомления, биометрический вход
- О приложении / условия / политика
- Выход

### M3. Гараж (мои авто)
- Список авто (карточки с фото, госномером, маркой/моделью)
- Empty state: «Добавьте первый автомобиль»
- Добавление авто: ввод госномера → **автозаполнение через NAPP** → подтверждение/коррекция марки/модели/года/VIN
- Загрузка фото техпаспорта (front/back)
- Карточка авто (детали, фото, документы)
- Удаление с подтверждением

### M4. Каталог продуктов
- Главная: 2 большие карточки — **ОСАГО** и **КАСКО**
- Сравнение продуктов (таблица: что покрывает, цена «от», срок)
- Подробная страница продукта: что входит, FAQ, кнопка «Рассчитать»

### M5. Калькулятор тарифа
- Шаг 1: выбор продукта (ОСАГО/КАСКО)
- Шаг 2: выбор авто из гаража или новое
- Шаг 3: водители (1 или несколько) — возраст, стаж
- Шаг 4: период страхования
- Шаг 5: дополнительные опции (для КАСКО — франшиза, угон/ущерб)
- Итог: цена + разбивка (госбаза + надбавки) + кнопка «Оформить»
- Состояние: «Пересчитать», «Сохранить расчёт»

### M6. Оформление полиса
- Резюме: продукт + авто + водители + период + цена
- Шаг 1: подтверждение данных страхователя
- Шаг 2: подтверждение данных авто
- Шаг 3: подтверждение водителей
- Шаг 4: оплата — выбор метода: **Uzcard (приоритет) / сохранённая карта / рассрочка** (опционально позже Click/Payme/Uzum)
- Экран оплаты (внешний WebView или нативный)
- Успех: «Полис выпущен!» + кнопки «Открыть QR» / «Скачать PDF»
- Ошибка: «Платёж не прошёл» с действиями

### M7. Платежи
- Список сохранённых карт (Uzcard / Humo / Visa)
- Добавление карты (через Uzcard прямой эквайринг)
- Удаление карты
- История платежей
- **Рассрочка:** список доступных планов (своя страховой + внешние BNPL — Uzum Nasiya / Alif / Anor — на согласовании)

### M8. Мои полисы
- Вкладки: Активные / Архив
- Карточка полиса (статус, авто, период, цена)
- Empty state
- Детали полиса:
  - Большая кнопка «Показать QR» (полноэкранный QR для инспектора ГАИ)
  - Скачать PDF
  - Условия и покрытие
  - Заявить о ДТП → переход в M9
  - **Вызвать аварийного комиссара** → быстрый action
  - Аннулировать полис → подтверждение → 100% возврат (без расчёта по дням)
  - Продлить → переход в M5/M6

### M9. Урегулирование убытков (заявление о ДТП)
- Шаг 1: тип события (ДТП с другим участником / повреждение / угон / др.)
- Шаг 2: гео-локация (карта + ручная корректировка)
- Шаг 3: дата и время
- Шаг 4: участники (количество, краткие данные)
- Шаг 5: фото 360° (минимум 8 ракурсов — макеты с подсказками)
- Шаг 6: фото повреждений (свободное число)
- Шаг 7: описание (текст + голосовая запись опционально)
- Шаг 8: **Электронный европротокол** (если применимо) — заполнение полей, подписи обоих участников (через ввод телефона + SMS-OTP двух участников)
- Шаг 9: **Опционально: вызов аварийного комиссара** (если ситуация сложная)
- Шаг 10: подтверждение и отправка
- Успех: «Заявление принято, № WX-12345»

### M9.1. Вызов аварийного комиссара (отдельный экран)
- Гео-локация (карта)
- Краткое описание
- Подтверждение вызова
- Статус: «Комиссар назначен, прибудет через X мин» + связь с комиссаром

### M10. Статус выплат
- Список открытых заявлений
- Карточка заявления: тайм-лайн (подано → рассмотрение → решение → выплата) + **таймер SLA** (60 дней default, переменно по тарифу)
- Детали: фото, описание, переписка с менеджером
- Чат внутри заявления

### M11. Уведомления (центр)
- Список уведомлений: окончание полиса, статус заявления, новости, акции, напоминание о записи к партнёру

- Empty state

### M12. Документы
- Список: договоры, чеки, акты
- Превью PDF в приложении
- Скачивание / шаринг

### M13. Поддержка

### M14. Главный экран приложения (после логина)

## 7. Универсальная дизайн-система

### Что нужно нарисовать как переиспользуемое
- **Кнопки:** primary / secondary / tertiary / destructive — состояния default/hover/pressed/disabled/loading. Отдельный стиль для **«SOS» / экстренных кнопок** (заявить ДТП, вызвать комиссара)
- **Поля ввода:** text / number / phone / OTP / select / date / file-upload / search — состояния default/focus/error/disabled
- **Карточки:** product / policy / vehicle / claim / **partner**
- **Чипы и теги:** статусы (активен / истёк / в работе / отклонён)
- **Модалки и нижние шторы (bottom sheets)**
- **Тосты / снекбары / алерты** — info / success / warning / error
- **Skeleton-лоадеры** для всех списочных экранов
- **Иконки:** единый набор (24px стандарт + 16px / 32px)
- **Иллюстрации:** для empty states, успехов, ошибок, онбординга
- **Навигация:** таб-бар (mobile), боковая панель (admin / partner), хедер/футер (web)
- **Темы:** **light + dark** обязательно для mobile

### Состояния, которые часто забывают
- Очень длинные имена / госномера / адреса (overflow)
- **Узбекский язык в латинице И кириллице** (длиннее русского, нужны более широкие кнопки/таб-бар; кириллица отличается от русской — например "ў", "қ", "ғ", "ҳ")
- Network-offline экран
- «Документ не загрузился»
- «Нет камеры / нет гео / нет интернета»
- «Полис скоро истекает» (за 30/14/3 дня)
- **«Партнёр временно недоступен»**

## 8. Пользовательские флоу (для прототипа)

### Флоу 1: Покупка ОСАГО (новый клиент)
Splash → Онбординг → Auth (телефон + OTP **или MyID/OneID**) → Главная → Каталог → ОСАГО → Калькулятор → Оформление → Оплата (Uzcard) → Успех → Полис с QR

### Флоу 2: Заявление о ДТП + европротокол
Главная → SOS-кнопка / Активный полис → «Заявить ДТП» → 10 шагов (включая электронный европротокол) → Отправка → Трекер статуса

### Флоу 3: Вызов аварийного комиссара
Главная → «Вызвать комиссара» → Гео + описание → Подтверждение → Статус «Комиссар в пути»

### Флоу 4: Продление полиса
Push «полис истекает» → Открытие приложения → Полис → «Продлить» → Калькулятор (предзаполнен) → Оплата → Новый полис

### Флоу 5: Запись к партнёру
Активный полис → Партнёры рядом → Карточка СТО → Выбор услуги → Дата/время → Подтверждение → Уведомление партнёру
