# SOS24 — Спецификация экранов мобильного приложения

> **Для Claude Design.**
> Дизайн-система и стилистика уже определены по двум референсным экранам.
> Этот документ — архитектура и структура каждого экрана. Дизайн-решения принимаются самостоятельно в рамках установленной стилистики.
>
> **Платформа:** iOS + Android (React Native)
> **Локали:** uz-Latn, uz-Cyrl, ru, en — все тексты на русском как placeholder, реальный перевод добавляется отдельно
> **Тема:** поддержка светлой и тёмной темы

---

## Навигация приложения (общая структура)

```
Splash Screen
    └── Onboarding (3 слайда) — только первый запуск
            ├── Регистрация (новый пользователь)
            │       └── Ввод номера → OTP → Данные профиля → Главная
            └── Вход (существующий пользователь)
                    └── Ввод номера → OTP → Главная

Главная навигация (Bottom Tab Bar — 4 таба):
    ├── Главная (Home)
    ├── Мои полисы (Policies)
    ├── Гараж (Garage)
    └── Профиль (Profile)

Модальные / стековые экраны (поверх таб-навигации):
    ├── Каталог продуктов → Калькулятор → Оформление → Оплата → Успех
    ├── Урегулирование убытков (заявление о ДТП)
    ├── Трекер выплат
    ├── Уведомления
    ├── Документы
    ├── Поддержка (чат)
    └── Партнёры (каталог СТО / медклиник)
```

## M1 — Онбординг + Авторизация

### M1.1 — Splash Screen
**Цель:** первый экран при запуске, брендинг, инициализация приложения

**Блоки сверху вниз:**
- Центр экрана: логотип SOS24 (большой)
- Под логотипом: слоган «Помощник на дороге 24/7»
- Нижняя часть: индикатор загрузки (тонкий прогресс или пульсация логотипа)

**Логика:**
- Показывается 1.5–2 сек пока грузится приложение
- Если первый запуск → переход на Onboarding
- Если уже авторизован → переход на Главную
- Если не авторизован → переход на экран Входа

---

### M1.2 — Онбординг (слайды 1–3)
**Цель:** показать ценность приложения новому пользователю

**Структура каждого слайда:**
- Верхняя часть (60% экрана): иллюстрация / анимация по теме слайда
- Заголовок слайда (крупный)
- Подзаголовок / описание (2–3 строки)
- Нижняя часть: индикатор шагов (точки 1/2/3) + кнопки

**Содержание слайдов:**

| Слайд | Иллюстрация | Заголовок | Описание |
|-------|-------------|-----------|----------|
| 1 | Авто + щит/полис | «Страховка за 5 минут» | Оформите ОСАГО или КАСКО онлайн, получите электронный полис мгновенно |
| 2 | Телефон + карта / аварийный комиссар | «Помощь на дороге» | Вызовите аварийного комиссара, оформите европротокол прямо с телефона |
| 3 | Сеть СТО / иконки партнёров | «Сеть партнёров» | СТО и медклиники рядом с вами — запись и оплата через приложение |

**Кнопки:**
- Слайды 1–2: «Далее» (основная) + «Пропустить» (текстовая ссылка, верхний правый угол)
- Слайд 3: «Начать» (основная, ведёт на экран Регистрации/Входа)

---

### M1.3 — Регистрация / Вход (выбор)
**Цель:** пользователь выбирает — войти или зарегистрироваться

**Блоки:**
- Логотип SOS24 (средний размер)
- Заголовок: «Добро пожаловать»
- Подзаголовок: «Войдите или создайте аккаунт»
- Кнопка «Войти» (основная)
- Кнопка «Зарегистрироваться» (вторичная / outline)
- Нижний текст: «Продолжая, вы соглашаетесь с условиями оферты и политикой конфиденциальности» (кликабельные ссылки)

---

### M1.4 — Ввод номера телефона
**Цель:** ввод номера для получения OTP

**Блоки:**
- Заголовок: «Введите номер телефона» (или «Вход» / «Регистрация» в зависимости от пути)
- Описание: «Отправим SMS с кодом подтверждения»
- Поле ввода:
  - Флаг + код страны (+998, Узбекистан) — зафиксирован, не меняется
  - Маска: +998 (XX) XXX-XX-XX
- Кнопка «Получить код» (основная, активна только при валидном номере)
- Состояния поля: пустое / фокус / заполнено / ошибка (неверный формат)

**Логика:** → переход на M1.5 (ввод OTP)

---

### M1.5 — Ввод OTP кода
**Цель:** подтверждение номера телефона

**Блоки:**
- Заголовок: «Введите код из SMS»
- Описание: «Код отправлен на +998 XX XXX-XX-XX» (номер подставляется)
- Поле OTP: 6 ячеек / боксов для цифр (большие, удобные для нажатия)
- Таймер обратного отсчёта: «Повторить через 01:30»
- После истечения таймера: ссылка «Отправить код повторно»
- Ссылка «Изменить номер» (возврат назад)

**Состояния:**
- Ввод: ячейки пустые / активная подсвечена
- Заполнено: автоматическая отправка (без нажатия кнопки)
- Ошибка: ячейки красные + текст «Неверный код, попробуйте снова»
- Успех: короткая анимация ✓ → переход дальше

**Логика:**
- Новый пользователь → M1.6 (заполнение профиля)
- Существующий пользователь → Главная (Home)

---

### M1.6 — Заполнение профиля (только при регистрации)
**Цель:** собрать основные данные нового пользователя

**Блоки:**
- Заголовок: «Расскажите о себе»
- Описание: «Нужно для оформления полисов»
- Прогресс: шаг 1 из 1 (или индикатор)
- Поля формы:
  1. Имя (текст)
  2. Фамилия (текст)
  3. Отчество (текст, необязательное)
  4. Дата рождения (date picker)
  5. Выбор языка интерфейса (uz-Latn / uz-Cyrl / ru / en — сегментед контрол или список)
- Кнопка «Продолжить» (основная, активна когда заполнены обязательные поля)

**Логика:** → переход на Главную (Home), первый вход

---

## M2 — Профиль пользователя

### M2.1 — Главный экран профиля
**Цель:** персональные данные, настройки, управление аккаунтом

**Блоки сверху вниз:**
- Хедер:
  - Аватар (инициалы или фото, если загружено)
  - Имя и фамилия
  - Номер телефона
  - Кнопка редактирования профиля (иконка карандаш или «Редактировать»)
- Секция «Мои документы»:
  - Строка: Паспорт (добавлен / не добавлен + статус верификации)
  - Строка: Водительское удостоверение (добавлено / не добавлено)
- Секция «Настройки»:
  - Строка: Язык интерфейса (текущий язык + стрелка)
  - Строка: Тема (Светлая / Тёмная / Системная)
  - Строка: Уведомления (вкл/выкл + стрелка к настройкам)
- Секция «Помощь»:
  - Строка: Поддержка
  - Строка: FAQ
  - Строка: О приложении (версия)
  - Строка: Оферта и политика конфиденциальности
- Кнопка «Выйти» (деструктивный стиль, красный текст)

---

### M2.2 — Редактирование профиля
**Цель:** изменить личные данные

**Блоки:**
- Хедер: «Редактировать профиль» + кнопка «Сохранить» (правый верхний угол)
- Аватар: фото с кнопкой «Изменить фото»
- Поля:
  1. Имя
  2. Фамилия
  3. Отчество (необязательное)
  4. Дата рождения
  5. Адрес (необязательное)
- Кнопка «Сохранить» (основная внизу, дублирует хедер)

---

### M2.3 — Добавление паспорта / водительского удостоверения
**Цель:** верификация документов для оформления полисов

**Блоки:**
- Заголовок: «Паспортные данные» (или «Водительское удостоверение»)
- Инфо-блок: зачем это нужно (1–2 строки)
- Поля ввода:
  - Серия и номер
  - Дата выдачи
  - Кем выдан
  - ПИНФЛ (для паспорта)
- Блок загрузки фото документа: плашка «Загрузить фото» (передняя + задняя стороны)
- Кнопка «Сохранить»
- Статус: ожидает проверки / проверено / отклонено (с причиной)

---

## M3 — Гараж (мои автомобили)

### M3.1 — Список автомобилей
**Цель:** хранение автомобилей пользователя, быстрый доступ при оформлении полиса

**Состояния:**

**Пустой список:**
- Иллюстрация (гараж / авто)
- Заголовок: «Добавьте свой автомобиль»
- Описание: «Автомобили сохраняются для быстрого оформления полисов»
- Кнопка «Добавить автомобиль» (основная)

**Есть автомобили:**
- Список карточек автомобилей:
  - Каждая карточка содержит:
    - Иконка / изображение авто (по марке если возможно, или универсальная)
    - Гос. номер (крупно)
    - Марка, модель, год
    - Активный полис: есть / нет (бейдж)
  - Свайп влево: действия «Редактировать», «Удалить»
- FAB кнопка «+» (добавить ещё авто) — правый нижний угол

---

### M3.2 — Добавление / редактирование автомобиля
**Цель:** ввод данных авто, автозаполнение через NAPP по гос. номеру

**Блоки:**
- Заголовок: «Добавить автомобиль»
- Поле «Гос. номер»:
  - Крупное поле с маской узбекского номера
  - Кнопка «Найти» / автопоиск после ввода
  - Состояния: поиск (лоадер), найдено (поля заполнились), не найдено (ручной ввод)
- Блок «Данные автомобиля» (заполняется автоматически или вручную):
  - Марка
  - Модель
  - Год выпуска
  - Цвет
  - VIN / кузов номер
  - Тип (легковой, грузовой, мотоцикл и др.)
  - Объём двигателя (куб. см)
  - Мощность (л.с.)
- Кнопка «Сохранить» (основная)

**Логика:**
- Автозаполнение данных от NAPP по гос. номеру
- Если NAPP не нашёл → пользователь вводит вручную
- При редактировании → поля предзаполнены

## M4 — Каталог продуктов

### M4.1 — Каталог страховых продуктов
**Цель:** выбор типа страхования

**Блоки:**
- Заголовок: «Страхование»
- Карточки продуктов (по одной или две в ряд):

  **Карточка ОСАГО:**
  - Название: «ОСАГО»
  - Расшифровка: «Обязательное страхование гражданской ответственности»
  - Ключевые преимущества: 3 пункта (иконка + текст)
    - Электронный полис мгновенно
    - Вся Узбекистан
    - Онлайн-оформление
  - Цена: «от X XXX сум / год»
  - Кнопка: «Рассчитать»

  **Карточка КАСКО:**
  - Название: «КАСКО»
  - Расшифровка: «Комплексное страхование автомобиля»
  - Ключевые преимущества: 3 пункта
    - Угон и ущерб
    - Вызов комиссара
    - Сеть партнёрских СТО
  - Цена: «от X XXX сум / год»
  - Кнопка: «Рассчитать»

- Промо-баннер (опционально): акции, специальные предложения

---

### M4.2 — Детальная страница продукта
**Цель:** подробное описание продукта перед расчётом

**Блоки:**
- Хедер: название продукта (ОСАГО / КАСКО)
- Раздел «Что покрывает»: список рисков с иконками и описанием
- Раздел «Что не покрывает»: исключения (кратко)
- Раздел «Как это работает»: шаги (иллюстрированные)
- Раздел «Вопросы и ответы»: аккордеон (FAQ)
- Кнопка «Рассчитать стоимость» (залипающая внизу)

## M5 — Калькулятор тарифа

### M5.1 — Калькулятор ОСАГО (пошаговый)
**Цель:** рассчитать стоимость полиса ОСАГО

**Структура:** stepper / wizard (шаги с прогресс-баром вверху)

**Шаг 1 — Автомобиль:**
- Заголовок: «Выберите автомобиль»
- Список авто из Гаража (если есть) — карточки с гос. номером и моделью
- Кнопка «Добавить новый автомобиль» (ведёт на M3.2)
- Если авто нет → сразу форма добавления

**Шаг 2 — Водители:**
- Заголовок: «Кто будет управлять»
- Переключатель: «Без ограничений» / «Ограниченный круг водителей»
- Если ограниченный: список водителей с кнопкой «Добавить водителя»
  - Добавление водителя: ФИО, серия/номер ВУ, дата выдачи, стаж
- Пояснение: как количество водителей влияет на стоимость

**Шаг 3 — Период страхования:**
- Заголовок: «Срок страхования»
- Варианты: 3 мес / 6 мес / 12 мес (сегментированный контрол или карточки)
- Дата начала: date picker (по умолчанию — завтра)
- Дата окончания: вычисляется автоматически

**Шаг 4 — Результат расчёта:**
- Заголовок: «Стоимость полиса»
- Карточка с итогом:
  - Базовая ставка
  - Коэффициенты (каждый коэффициент на строке: название + значение + сумма влияния)
  - Итого: сумма (крупно)
- Вариант оплаты: Единовременно / Рассрочка (переключатель)
  - Рассрочка: показать кол-во платежей и сумму каждого
- Кнопка «Оформить полис» (основная)
- Кнопка «Пересчитать» (вторичная)

---

### M5.2 — Калькулятор КАСКО
**Цель:** рассчитать стоимость КАСКО

**Шаги аналогичны ОСАГО, добавляются:**

**Шаг — Страховая сумма:**
- Заголовок: «Стоимость автомобиля»
- Поле: сумма в сумах (или USD — уточнить с андеррайтером)
- Пояснение: рыночная / договорная стоимость

**Шаг — Опции покрытия (чекбоксы):**
- Угон
- Полная гибель
- Ущерб от ДТП
- Стихийные бедствия
- Противоправные действия третьих лиц
- Медицинские расходы пассажиров

## M6 — Оформление полиса (Чекаут)

### M6.1 — Экран оформления
**Цель:** финальная проверка данных перед оплатой

**Блоки:**
- Хедер: «Оформление полиса»
- Секция «Автомобиль»: гос. номер, марка/модель — с кнопкой «Изменить»
- Секция «Водители»: список водителей — с кнопкой «Изменить»
- Секция «Период»: дата начала и окончания — с кнопкой «Изменить»
- Секция «Стоимость»:
  - Итоговая сумма (крупно)
  - Способ оплаты
- Чекбокс: «Я ознакомился с условиями оферты и согласен» (обязательный)
- Кнопка «Перейти к оплате» (основная, залипающая внизу)

## M7 — Платежи

### M7.1 — Экран оплаты
**Цель:** ввод данных карты и совершение оплаты

**Блоки:**
- Хедер: «Оплата»
- Сумма к оплате (крупно, по центру)
- Описание: «ОСАГО · [марка авто] · [период]»
- Секция «Способ оплаты»:
  - Сохранённые карты (если есть): список карточек с логотипом (Uzcard / Humo), последние 4 цифры
  - Опция «Новая карта»
- Форма новой карты (если выбрана):
  - Поле: Номер карты (маска: XXXX XXXX XXXX XXXX)
  - Поле: Срок действия (ММ/ГГ)
  - Чекбокс: «Сохранить карту»
- Секция «Рассрочка» (если выбрана ранее):
  - Инфо: «X платежей по X XXX сум»
  - Первый платёж: X XXX сум (сегодня)
- Кнопка «Оплатить X XXX сум» (основная)

**Состояния:**
- Обработка платежа: лоадер / блокировка экрана
- Успех → M7.2
- Ошибка: сообщение + кнопка «Попробовать снова»

---

### M7.2 — Успешная оплата
**Цель:** подтверждение оплаты и получения полиса

**Блоки:**
- Анимация успеха (галочка / конфетти)
- Заголовок: «Полис оформлен!»
- Описание: «ОСАГО действует с [дата] по [дата]»
- Карточка полиса (мини-превью):
  - Номер полиса
  - Автомобиль
  - QR-код (маленький, кликабельный для увеличения)
- Кнопки:
  - «Скачать PDF» (вторичная)
  - «Мои полисы» (основная)
  - «На главную» (текстовая ссылка)

---

### M7.3 — Управление картами
**Цель:** список сохранённых карт, добавление/удаление

**Блоки:**
- Заголовок: «Мои карты»
- Список карт:
  - Карточка: логотип (Uzcard/Humo) + последние 4 цифры + срок действия
  - Кнопка удаления (свайп или кнопка)
- Кнопка «Добавить карту» (вторичная)

---

## M8 — Мои полисы

### M8.1 — Список полисов
**Цель:** все полисы пользователя в одном месте

**Состояния:**

**Пустой список:**
- Иллюстрация
- Заголовок: «У вас пока нет полисов»
- Кнопка «Оформить полис» (основная)

**Есть полисы:**
- Фильтр/табы: «Активные» / «Архив»
- Список карточек полисов:
  - Каждая карточка:
    - Тип: ОСАГО / КАСКО (бейдж/цвет)
    - Статус: Активен / Истекает (предупреждение) / Истёк (серый)
    - Автомобиль: гос. номер + марка
    - Период: «01.01.2026 — 31.12.2026»
    - Номер полиса
    - Кнопка «QR-код» (быстрый доступ)
  - Нажатие на карточку → M8.2 (детали полиса)

---

### M8.2 — Детали полиса
**Цель:** полная информация о полисе, действия

**Блоки:**
- Хедер: тип полиса + статус (бейдж)

### M9.3 — Электронный европротокол
**Цель:** пошаговое оформление европротокола

**Структура:** wizard (прогресс-бар)

**Шаг 1 — Обстоятельства ДТП:**
- Дата и время (по умолчанию — сейчас)
- Место ДТП (адрес из геолокации + редактирование)
- Кол-во транспортных средств (2 — обязательно для европротокола)
- Пострадавшие: Да / Нет (если «Да» → блокируем европротокол, предлагаем вызов комиссара)

**Шаг 2 — Транспортные средства:**
- Секция «Моё ТС»: подтягивается из профиля / гаража
- Секция «Второй участник»:
  - Гос. номер (ввод)
  - ФИО водителя
  - Страховой полис (серия, номер)
  - Телефон

**Шаг 3 — Схема ДТП:**
- Поле для рисования схемы столкновения (простой canvas)
- Или: выбор из готовых шаблонов-схем (сетка вариантов)
- Описание обстоятельств (текстовое поле)

**Шаг 4 — Фотофиксация:**
- Инструкция: что фотографировать (иконки: общий вид, повреждения, номерные знаки, документы)
- Кнопки загрузки для каждого типа фото:
  - Место ДТП (общий план) — обязательно
  - Повреждения моего авто — обязательно
  - Повреждения второго авто — обязательно
  - Документы второго участника — опционально
- Статус загрузки каждого фото

**Шаг 5 — Подписание:**
- Итоговый просмотр данных (всё заполненное)
- Информация об электронной подписи
- Чекбокс: «Подтверждаю, что все данные верны»
- Кнопка «Подписать и отправить»

**После отправки:**
- Экран успеха: номер заявления + переход к трекеру (M10)

## M10 — Статус выплат

### M10.1 — Список заявлений

**Состояния:**

**Есть заявления:**

      - Одобрено (зелёный)

### M10.2 — Детали заявления / Трекер
**Цель:** детальное отслеживание статуса выплаты

**Блоки:**

- Блок «Детали случая»:
  - Тип случая
  - Дата события

- Блок «Документы по делу»: список загруженных фото/документов
- Блок «Комментарии оператора» (если есть)

- Кнопка «Связаться с оператором» → чат поддержки (M13)

### Хедеры экранов
- Большинство экранов: заголовок по центру + кнопка «Назад» слева

### Общие компоненты (используются везде)
- Кнопка Primary (основная): полная ширина, скруглённая
- Кнопка Secondary (вторичная): outline или ghost
- Кнопка Destructive (деструктивная): красная
- Input (поле ввода): лейбл сверху, поле, хинт снизу, состояния
- Card (карточка): скруглённые углы, тень или граница
- Badge (бейдж): статус, маленький тег
- Modal / Bottom Sheet: всплывающее снизу
- Toast / Snackbar: уведомление снизу (успех, ошибка, инфо)
- Skeleton Loader: анимированный плейсхолдер при загрузке
